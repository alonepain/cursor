# Руководство по диагностике проблем в приложениях

## 🔍 Общие шаги диагностики

### 1. Определите тип проблемы
- **Приложение не запускается** - проверьте зависимости и конфигурацию
- **Приложение зависает** - проверьте логи и производительность
- **Ошибки во время работы** - анализируйте сообщения об ошибках
- **Проблемы с производительностью** - профилируйте приложение

### 2. Проверьте логи
```bash
# Для Node.js приложений
npm run dev
# или
yarn dev

# Для Python приложений
python app.py
# или
python -m flask run

# Системные логи Linux
journalctl -f
tail -f /var/log/syslog
```

### 3. Проверьте зависимости
```bash
# Node.js
npm install
npm audit
npm outdated

# Python
pip install -r requirements.txt
pip check

# Системные пакеты
sudo apt update
sudo apt upgrade
```

### 4. Проверьте порты и сетевые подключения
```bash
# Проверить какие порты заняты
netstat -tulpn
ss -tulpn

# Проверить конкретный порт
lsof -i :3000
```

### 5. Проверьте переменные окружения
```bash
# Посмотреть все переменные
env

# Проверить конкретную переменную
echo $NODE_ENV
echo $DATABASE_URL
```

## 🐛 Специфичные проблемы

### Web-приложения
1. **Проверьте браузерную консоль** (F12)
2. **Очистите кэш браузера** (Ctrl+Shift+R)
3. **Проверьте Network tab** на наличие ошибок загрузки
4. **Проверьте CORS настройки**

### База данных
```bash
# Проверить подключение к базе
ping database-host
telnet database-host 5432

# Проверить статус службы
systemctl status postgresql
systemctl status mysql
```

### API и микросервисы
```bash
# Проверить health check
curl http://localhost:3000/health
curl -I http://localhost:3000/api

# Проверить статус с подробностями
curl -v http://localhost:3000/api/endpoint
```

## 🛠️ Инструменты диагностики

### Мониторинг системы
```bash
# Использование CPU и памяти
top
htop
free -h
df -h

# Процессы приложения
ps aux | grep "app-name"
pgrep -f "app-name"
```

### Сетевая диагностика
```bash
# Проверить DNS
nslookup domain.com
dig domain.com

# Проверить соединение
ping google.com
traceroute google.com
```

## 🆘 Когда обращаться за помощью

Приложите к вопросу:
1. **Полный текст ошибки** из логов
2. **Версии** используемых технологий
3. **Шаги для воспроизведения** проблемы
4. **Что изменилось** перед появлением проблемы
5. **Системную информацию** (OS, версии)

## 🔧 Полезные команды для сбора информации

```bash
# Информация о системе
uname -a
lsb_release -a
cat /etc/os-release

# Версии языков программирования
node --version
python --version
php --version
java -version

# Информация о Docker (если используется)
docker --version
docker-compose --version
docker ps
docker logs container-name
```
